End-to-End Procure-to-Pay (P2P) Flow

Introduction to Oracle Fusion Cloud Procurement

This document outlines the end-to-end Procure-to-Pay (P2P) process within Oracle Fusion Cloud. The P2P cycle is a critical business process that encompasses all activities from identifying a need for goods or services to making the final payment to the supplier. In Oracle Fusion Cloud, this process is streamlined and automated to enhance efficiency, ensure compliance, and provide real-time visibility into procurement activities. This flow covers key phases including requisitioning, purchase order management, goods receipt, invoice processing, and final payment, culminating in integration with the General Ledger.

üßæ 4. Invoice Processing

Suppliers send invoices, which are matched against POs and receipts.

P2P-32 Create Invoice Matched to a Purchase Order ‚ñº

Navigator: Payables > Invoices > Tasks > Invoices Sub-Category > Create Invoices

  • Search Invoices for PO
  • Click on Desired PO and all information to fill in from default
  • Enter Invoice Number for Number
  • Enter Invoice Amount for Amount
  • Select Invoice Type
  • Enter Invoice Date (default is current date)
  • Select Payment Terms
  • Enter Terms data (default is current date)
  • Save
  • In Lines section, click the Go icon next to Match Invoice Lines
  • Check Match box next to line
  • Alter Amount if not paying in full
  • Apply
  • OK
  • Invoice Line amount, PO Number, PO Line and PO Schedule gets defaulted from the PO
  • Save
  • Invoice Actions > Validate
  • Once Validated, Save and Close
P2P-36 Run Create Accounting for Invoices ‚ñº

Navigator: Payables > Tasks > Accounting sub-category > Create Accounting

  • Subledger Application: Payables
  • Ledger: City LEDGER
  • Report Style: Detail
  • Submit
  • Ok to confirm
  • Click Cancel to Exit
  • To Validate Process: Navigator > Tools > Scheduled Processes > Refresh > Check Status (Succeeded)
